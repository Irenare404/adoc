INSTALLATION

安装与启动

  1. 1
    关闭服务器

    不要在运行中的服务器直接替换插件 JAR。

  2. 2
    安装插件

    将 AusDungeon、MythicMobs 和其他依赖放入 plugins/。

  3. 3
    启动 Paper

    首次启动生成 plugins/AusDungeon/ 配置目录。

  4. 4
    检查日志

    控制台应出现 AusDungeon enabled with 1 dungeon(s).

PowerShell
java -Xms1G -Xmx2G -jar paper-1.21.8-60.jar --nogui
03

QUICK START

十分钟创建首个副本

以下流程创建一个名为 ruins 的两阶段副本。先在模板世界完成地图，再按玩家当前位置记录配置。

01创建并选中副本/ad create ruins
02设置等待点与开局点/ad set lobby/ad set spawn
03创建阶段/ad stage add wave1
04设置触发区域/ad set pos1/ad stage setregion wave1
05登记刷怪/ad stage addspawn wave1 center/ad stage addmob wave1 SkeletonKingMinion 3 center
06登记障碍与奖励/ad stage addbarrier wave1/ad reward addcmd give {player} diamond 3
07持久化/ad save
区域设置方式

在第一个角执行 /ad set pos1，移动到对角后执行 /ad stage setregion <stageId>。也可先执行 /ad set pos2 检查选点；/ad pos1 仍可作为简写。当前版本不直接读取 WorldEdit 选区。

WORLD MODEL

模板与实例世界

配置坐标属于模板世界。每次创建实例，插件会复制模板世界目录并加载一个独立世界。

  • 模板世界必须已加载
  • 实例结束后自动卸载并删除
  • 不应在实例世界保存永久数据
  • 使用小型专用世界降低复制延迟

命令参考

06

ACCESS CONTROL

权限节点

ausdungeon.use玩家功能

打开菜单、加入和退出副本。默认所有玩家拥有。

ausdungeon.admin管理功能

编辑配置、测试、强制推进和终止实例。默认 OP 拥有。

07

RUNTIME

副本生命周期

WAITING

等待玩家，可继续加入。

COUNTDOWN

满足最低人数，倒计时中。

RUNNING

实例锁定，阶段推进中。

COMPLETED

Boss/末阶段完成，发放奖励。

FAILED

全员死亡、退出或管理员终止。

CLOSING

传送玩家、卸载并删除世界。

死亡与退出

死亡玩家进入旁观模式，不再参与区域全员判定。可使用 /ad leave 立即退出。通关或失败后默认等待 5 秒关闭，期间同样可主动退出。

DUNGEON SCHEMA

dungeons.yml

最小完整示例
dungeons:
  ancient_ruins:
    display-name: "Ancient Ruins"
    description: "A two-stage example dungeon"
    min-players: 1
    max-players: 4
    countdown-seconds: 60
    lobby: { world: dungeon_world, x: 0.5, y: 65, z: 0.5, yaw: 0, pitch: 0 }
    spawn: { world: dungeon_world, x: 5.5, y: 65, z: 0.5, yaw: 0, pitch: 0 }
    stages:
      courtyard:
        display-name: "Ruined Courtyard"
        trigger-region:
          world: dungeon_world
          min: { x: 10, y: 60, z: -5 }
          max: { x: 20, y: 75, z: 5 }
        spawn-points:
          center: { world: dungeon_world, x: 15.5, y: 65, z: 0.5 }
        mobs:
          - { mythic-mob-id: SkeletonKingMinion, amount: 3, spawn-point: center, delay-ticks: 0 }
        barriers:
          0: { world: dungeon_world, x: 22, y: 65, z: 0 }
        next-stage: throne_room
        boss-stage: false
      throne_room:
        display-name: "Throne Room"
        trigger-region:
          world: dungeon_world
          min: { x: 25, y: 60, z: -5 }
          max: { x: 40, y: 75, z: 5 }
        spawn-points:
          boss: { world: dungeon_world, x: 35.5, y: 65, z: 0.5 }
        mobs:
          - { mythic-mob-id: SkeletonKing, amount: 1, spawn-point: boss }
        barriers: {}
        boss-stage: true
    rewards:
      commands:
        - "give {player} diamond 3"
trigger-region

所有在线、非旁观实例玩家全部进入后触发。

mobs

只统计由本插件登记的 MythicMob UUID。

barriers

清怪后直接将这些坐标替换为 AIR。

boss-stage

完成该阶段后直接通关并结算。

end-story

清怪后固定玩家并按间隔向副本聊天框播放剧情，结束后再开路。

MESSAGING

messages.yml

消息使用 MiniMessage 格式，并且仅发送给当前实例玩家。支持聊天栏、ActionBar 与 Title / Subtitle。

MiniMessage
countdown: "<gold>Dungeon starts in <yellow>{seconds}</yellow> seconds.</gold>"
stage-complete: "<green>Stage cleared: {stage}</green>"
failed-subtitle: "<gray>Reason: {reason}</gray>"
{player}{current}{max}{seconds}{stage}{reason}

INTEGRATION

MythicMobs 对接

AusDungeon 只负责“何时、何地、生成多少只”。怪物技能、属性、装备和掉落都在 MythicMobs 中定义。

  • ID 区分大小写
  • 先使用 MythicMobs 命令验证怪物可生成
  • 只有 AD 登记的怪物死亡会推进阶段
验证怪物
/mm mobs spawn SkeletonKing 1

TROUBLESHOOTING

故障排查

点击菜单提示世界不可用

确认 dungeons.yml 中的模板世界已加载。使用 Multiverse-Core 导入后再测试。

全员进入区域但没有刷怪

确认 Y 高度、当前阶段、刷怪点以及 MythicMob ID。先用 /mm mobs spawn 单独验证。

清怪后障碍没有消失

确认所有本阶段登记怪物都已死亡，并检查障碍坐标属于当前模板世界。

通关后实例仍存在

检查奖励命令语法。当前版本会隔离奖励异常并继续清理；终态实例不会出现在活动实例补全中。

死亡后无法重新进本

执行 /ad leave 清除实例绑定、恢复生存模式并返回主世界。

15

SCOPE

一期限制

  • 仅支持线性阶段，不包含分支路线。
  • 游戏内编辑器覆盖基础字段，Boss 标记等高级项需编辑 YAML。
  • 不包含评分、评级、奖励箱品质与断线重连。
  • 实例世界使用文件复制，大型模板世界会增加创建延迟。
  • WorldEdit / WorldGuard 是兼容依赖，区域编辑由 AD 自己完成。
